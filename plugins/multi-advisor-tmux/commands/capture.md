# Capture

指定したペインの完了確認、または回答内容の取得を行います。

## 推奨方法

### 完了確認は /check コマンド
```bash
/multi-advisor-tmux:check 1
```
pane-status-checkerサブエージェント（Haiku）が自動で状態を判定します。
- `Ready` - 入力待ち（処理完了）
- `Processing` - 処理中
- `Awaiting Permission` - ユーザー操作待ち
- `Error` - エラー発生

### 回答取得は response-extractor
response-extractorエージェントを使用します。
「ペイン1の回答を取得して」と指示してください。

クリーンな回答のみを抽出し、コンテキストを節約します。

## メリット

- **コンテキスト節約**: サブエージェント（Haiku）を使用するため、メインのコンテキストを消費しない
- **確実な判定**: 40行を自動でキャプチャし、処理中表示を正確に検出
- **簡潔な結果**: 状態のみを1行で返すため、結果の確認が簡単
